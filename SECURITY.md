# Security Policy

## Supported Versions

Kinal is under active development.  
Security fixes are applied to the latest version.

## Reporting a Vulnerability

If you discover a security vulnerability in Kinal, please **do not open a public issue**.

Instead, report it privately via:

- Email: ccbabyqwq@outlook.com
- Or GitHub Security Advisories (preferred)

Please include:

- Description of the issue
- Steps to reproduce
- Potential impact
- A minimal proof of concept if possible

## Scope

Security issues include (but are not limited to):

- Memory safety issues
- Undefined behavior in compiler or VM
- Code execution vulnerabilities
- Sandbox / isolation bypass (if applicable)
- FFI-related unsafe behavior

## Response

We aim to:

- Acknowledge reports within a few days
- Investigate and fix as soon as possible
- Credit the reporter if desired
