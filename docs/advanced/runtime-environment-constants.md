# IO.Runtime (Execution Backend Constants)

`IO.Runtime.*` is a set of compile-time constants used in source code to determine whether the current artifact's execution backend is LLVM native compilation or KinalVM bytecode.

Like `IO.Target.*` and `IO.Host.*`, these symbols are folded directly to literal values during the semantic phase, producing no runtime overhead. Dead branches are eliminated normally by the backend.

> **Note:** `IO.Runtime` describes the execution backend (native / vm), not the runtime library mode. `--runtime none/alloc/gc` controls the memory allocation strategy; these are independent dimensions.

---

## Constants at a Glance

### Backend Identifier

The following symbols all return the `int` value for the current backend:

| Symbol | Description |
|--------|-------------|
| `IO.Runtime` | Current backend, equivalent to `IO.Runtime.Current` |
| `IO.Runtime.Current` | Same as above |
| `IO.Runtime.Kind` | Same as above |
| `IO.Runtime.Kind.Current` | Same as above |

Enum values:

| Symbol | Value | Description |
|--------|-------|-------------|
| `IO.Runtime.Unknown` | `0` | Unknown |
| `IO.Runtime.Native` | `1` | LLVM native compilation |
| `IO.Runtime.VM` | `2` | KinalVM bytecode |

`IO.Runtime.Kind.Native` / `IO.Runtime.Kind.VM` / `IO.Runtime.Kind.Unknown` are prefixed aliases with identical values.

### String Constants

| Symbol | Type | Description |
|--------|------|-------------|
| `IO.Runtime.Name` | `string` | Backend name, `"native"` or `"vm"` |

### Boolean Aliases

| Symbol | Type | Equivalent Expression |
|--------|------|-----------------------|
| `IO.Runtime.IsNative` | `bool` | `IO.Runtime == IO.Runtime.Native` |
| `IO.Runtime.IsVM` | `bool` | `IO.Runtime == IO.Runtime.VM` |

---

## Usage Example

```kinal
Unit MyLib;

Get IO.Console;

Static Function void PrintBackend()
{
    If (IO.Runtime.IsVM)
    {
        IO.Console.PrintLine("running on KinalVM");
    }
    Else
    {
        IO.Console.PrintLine("native build");
    }
}
```

Integer comparison is equally valid:

```kinal
If (IO.Runtime == IO.Runtime.VM)
{
    // ...
}
```

## Values by Command

| Command | `IO.Runtime` |
|---------|-------------|
| `kinal build` | `Native` |
| `kinal run` | `Native` |
| `kinal vm build` | `VM` |
| `kinal vm pack` | `VM` |
| `kinal vm run <file.kn>` | `VM` |
| `--emit obj` / `--emit asm` / `--emit llvm-ir` | `Native` |
| `--emit knc` | `VM` |

String values for `IO.Runtime.Name`:

| Command | `IO.Runtime.Name` |
|---------|------------------|
| `kinal build` / `kinal run` | `"native"` |
| `kinal vm build` / `kinal vm run` / `--emit knc` | `"vm"` |

Constants are written into the artifact at compile time and cannot be changed afterward. In a `.knc` file generated by `kinal vm build`, `IO.Runtime` is always `VM` regardless of how it is subsequently executed.

---

## Comparison with IO.Target / IO.Host

All three constant groups are folded at the semantic phase, differing in origin and purpose:

| Constant Group | Dimension Described | Typical Use |
|----------------|--------------------|-----------:|
| `IO.Target.*` | Target platform (OS, architecture, pointer width…) | Platform-conditional compilation, conditional linking |
| `IO.Host.*` | Host machine (the machine running the compiler) | Distinguishing host from target during cross-compilation |
| `IO.Runtime.*` | Execution backend (native / vm) | Library code providing fallback implementation for VM |

`IO.Runtime` is unrelated to `--runtime none/alloc/gc`; the latter describes the runtime library's memory allocation strategy and does not affect the value of `IO.Runtime`.
